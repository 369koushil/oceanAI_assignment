E-SHOP CHECKOUT - UI/UX GUIDELINES
Version: 1.0
Last Updated: Jan 2025

================================
VISUAL DESIGN STANDARDS
================================
Color Palette
-------------
Primary Gradient: #667eea → #764ba2  
Used for header, primary buttons, section titles.

Success Color: #28a745  
Used for: Pay Now button, success message.

Error Color: #dc3545  
Used for: All inline form validation errors, discount error messages.

Neutral Colors:
- Light Gray: #f8f9fa  
- Border: #e9ecef / #dddddd  
- Text Primary: #333  
- Text Secondary: #666

Typography
----------
Font Family: Segoe UI / Tahoma / Geneva / Verdana  
Sizes:
- H1: 2.5em  
- H2: 1.5em  
- Body: 1em  
- Error Text: 0.9em  
Weights:
- Titles: bold  
- Labels: 600  
- Body: 400

================================
COMPONENT SPECIFICATIONS
================================

Buttons
-------
Primary Button ("Add to Cart"):
- Background: #667eea (purple)
- Hover: #5568d3
- Text: white
- Full width in product cards

Pay Now Button:
- MUST be green (#28a745)
- Font-size: 1.2em
- Hover: #218838
- Disabled after success
- Text changes to “Payment Complete”

Form Elements
-------------
Input fields:
- Border: 2px solid #ddd
- Border-radius: 5px
- Padding: 12px
- Focus state: border color -> #667eea
- No outline

Textareas:
- Same as inputs
- Row count: 3

Radio Groups:
- White background
- Border: 2px solid #ddd (hover: #667eea)
- Border radius: 5px
- Padding: 12px

Validation Error Messages
-------------------------
Critical requirements:
- MUST be displayed in red (#dc3545)
- Font size: 0.9em
- Appears directly BELOW the field
- Only visible when error present
- Default state: hidden

Errors:
- Full Name: "Please enter your full name"
- Email: "Please enter a valid email address"
- Address: "Please enter your shipping address"
- Discount Code:
  * "Invalid discount code"
  * "Please enter a discount code"

================================
LAYOUT & INTERACTIONS
================================

Two-column layout:
- Left: Products + Cart
- Right: Form + Payment
- Single column on <768px

Product Cards:
- Hover: lift & shadow effect

Cart:
- Quantity field: input[type=number]
- Auto-updates totals

Success Message:
- Green (#28a745)
- Slide-down animation
- Shows: "✅ Payment Successful! Thank you for your order."

================================
CRITICAL DESIGN RULES
================================
1. Pay Now button MUST be green  
2. All error messages MUST be red  
3. Success message MUST be green  
4. Validation must be visible and below field  
5. Hover states required  
6. Required fields must show *  
7. Labels must be visible, not placeholders-only
